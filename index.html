Connect your data types — every record is validated

Each team uses different systems. Connecting runs a validation gate: IDs present · link resolves · mandatory fields. Clean records are trusted; anything that fails is held for review — it never silently enters the graph.

Review queue

Records that failed validation (missing or ambiguous links) wait here. ALK's IDs are not fully standardised, so this is expected — resolve each before it is trusted.

Digitise paper / manual records (OCR/AI)

Upload a scanned document, or try a sample. Clean scans ingest after a quick check; poor scans are held for human review. Paper uses OCR confidence; connectors use the checks above — same gate, different signal.
